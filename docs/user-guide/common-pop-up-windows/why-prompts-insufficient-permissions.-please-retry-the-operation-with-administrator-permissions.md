# Why prompts "Insufficient permissions. Please retry the operation with administrator permissions''?

<div align="center"><figure><img src="/files/CfWeCSwNGpnHyoLnnLni" alt=""><figcaption><p>Prompts when downloading assets</p></figcaption></figure></div>

<div align="center"><figure><img src="/files/XG2T1Bp09YQasoZVeUux" alt="" width="545"><figcaption><p>Prompts when downloading Demo scenes</p></figcaption></figure></div>

In 2.5 version, the installation experience has been optimized and the workspace has been forced to move out (Users can also customize the workspace directory during the installation). This error will be displayed if the current Windows account does not have relevant editing permissions for this workspace directory.&#x20;

Suggestion: If this prompt is displayed, you can open the current workspace location to check whether you have permission to create new documents in this path; if you don't have the relevant permission, you can run D5 as an administrator or \ <mark style="background-color:blue;">-</mark> <mark style="background-color:blue;"></mark><mark style="background-color:blue;">**\[Versions earlier than 2.11]:**</mark> migrate the workspace to a location with editing permissions in the preference settings.\ <mark style="background-color:blue;">**- \[Versions 2.11 and later]:**</mark> migrate the workspace to a location with editing permissions in D5 Launcher-Setting.
